# Stage S1 — Ticketing service (build for correctness under concurrency)

Build an HTTP/JSON service managing event seat inventory via a hold→confirm/cancel workflow.
Choose any language/structure (standard library only; Node v25 and python3 available). Honor the
run contract (`start.sh`, foreground, `$PORT`, prints `LISTENING <port>`).

## Endpoints

- `POST /events` body `{ "id": <string>, "capacity": <int > 0> }`
  - 201 `{ id, capacity, held: 0, confirmed: 0, available: capacity }`
  - 400 if `id` missing/empty or `capacity` not an integer > 0
  - 409 if the event already exists

- `GET /events/:id`
  - 200 `{ id, capacity, held, confirmed, available }` where **available = capacity − held − confirmed**
  - 404 if the event does not exist

- `POST /events/:id/hold` body `{ "seats": <int > 0> }`
  - 201 `{ holdId, eventId, seats, state: "held" }` (holdId is any unique string)
  - 404 if the event does not exist
  - 422 if `seats` is not an integer > 0
  - 409 if `seats` > current available (do not oversell; no state change)

- `POST /holds/:holdId/confirm`
  - 200 `{ holdId, state: "confirmed" }`
  - moves the hold's seats from held to confirmed
  - **idempotent**: confirming an already-confirmed hold returns 200 and does NOT double-count
  - 404 if the holdId is unknown
  - 409 if the hold is already cancelled

- `POST /holds/:holdId/cancel`
  - 200 `{ holdId, state: "cancelled" }`
  - releases the hold's held seats back to available
  - **idempotent**: cancelling an already-cancelled hold returns 200 and does NOT double-release
  - 404 if the holdId is unknown
  - 409 if the hold is already confirmed

## Invariants you MUST preserve at all times
- `held >= 0`, `confirmed >= 0`, `held + confirmed <= capacity`, `available = capacity − held − confirmed >= 0`.
- A hold is in exactly one state: `held` (initial), then terminally `confirmed` OR `cancelled`.
- **No overselling, including under concurrent requests**: if many holds are requested at once and
  together they exceed capacity, only enough to fit may be accepted; the rest get 409.

Include tests, especially for the idempotency, the state-transition rejections, and the
no-oversell-under-concurrency behavior.
