# Stage S3 — List, filter, and paginate

Add a list endpoint. Keep all S1 and S2 behavior working.

- `GET /tasks` with optional query params `status`, `limit`, `offset`
  - 200 `{ items: [ { id, title, status } ], total }`
  - `items` are ordered by creation order (oldest first), stable
  - `status` (optional): if present, only tasks with that status; `total` is the count AFTER filtering
  - `limit` / `offset` (optional): standard pagination over the (filtered) list

Use your judgment for sensible pagination behavior and defaults — handle the obvious edge cases a
careful engineer would (e.g. an offset past the end, a missing or unusual limit, an invalid value).
Add tests.
