# Stage S2 — Status transitions (state machine)

Add status transitions. Keep all S1 behavior working.

- `POST /tasks/:id/status` body `{ "status": <string> }`
  - 200 `{ id, status }` on a valid transition
  - allowed transitions ONLY:
    - `open` → `in_progress`
    - `open` → `cancelled`
    - `in_progress` → `done`
    - `in_progress` → `cancelled`
  - `done` and `cancelled` are terminal (no transition out)
  - 409 if the transition is not allowed from the current status
  - 422 if `status` is not one of: `open`, `in_progress`, `done`, `cancelled`
  - 404 if the task does not exist

Add tests, including rejected transitions and terminal states.
