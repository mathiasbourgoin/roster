# Stage S2 — Feature A: deposits

Add deposits to the existing service. Keep all S1 behavior working.

- `POST /accounts/:id/deposit` with body `{ "amount": <number> }`
  - 200 with `{ "id", "balance" }`, balance increased by `amount`
  - 404 if the account does not exist
  - 422 if `amount` is not a number > 0 (balance unchanged)

After any sequence of deposits, `GET /accounts/:id` must report `balance` equal to the
sum of successful deposits. Add tests for deposits.
