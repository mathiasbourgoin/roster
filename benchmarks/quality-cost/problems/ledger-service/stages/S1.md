# Stage S1 — Scaffold the ledger service

Build a small HTTP/JSON service that manages money accounts. Pick whatever language,
framework, and project structure you think is best — that choice is yours.

Honor the run contract (see `problem.json`): startable with a `PORT` env var, print
`LISTENING <port>` once bound, serve on `127.0.0.1:$PORT`.

Endpoints for this stage:

- `POST /accounts` with body `{ "id": "<string>" }`
  - 201 with `{ "id", "balance": 0 }` on success
  - 400 if `id` is missing/empty
  - 409 if the account already exists
- `GET /accounts/:id`
  - 200 with `{ "id", "balance" }`
  - 404 if the account does not exist

Set up the project so later features are easy to add. Include tests.
