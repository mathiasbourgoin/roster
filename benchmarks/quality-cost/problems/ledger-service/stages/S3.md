# Stage S3 — Feature B: withdrawals (similar to deposits)

Add withdrawals. This is deliberately similar in shape to deposits — a well-factored
S2 should make this cheap and safe. Keep all S1 and S2 behavior working.

- `POST /accounts/:id/withdraw` with body `{ "amount": <number> }`
  - 200 with `{ "id", "balance" }`, balance decreased by `amount`
  - 404 if the account does not exist
  - 422 if `amount` is not a number > 0 (balance unchanged)
  - 422 if the account has insufficient funds — **balance must never go negative**
    (balance unchanged)

Add tests for withdrawals, including the insufficient-funds case.
