# Iteration 2026-05-22-23: Dashboard Roster Metadata

## Brief

Make the dashboard visibly roster-native. The prior dashboard rows showed only
the raw `roster_agent` id; this loop enriches dashboard agents from
`index.json` so the TUI can show human-readable role/domain/source/tag hints
without coupling socket snapshots to local roster files.

## Acceptance Criteria

- Extend `Roster_index` to parse display name, description, domain, tags, path,
  and source for agent entries.
- Add optional roster metadata to dashboard agents.
- Keep dashboard rendering usable when no roster index is provided.
- Add a dashboard enrichment function that can be applied after local or socket
  model creation.
- Add `--roster-index` support to `tactl dashboard render`.
- Add `--roster-index` support to `tactl dashboard render-socket`.
- Keep compact rows readable at common widths and move full metadata into the
  selected preview.
- Cover parser, model, state CLI, and socket CLI behavior with tests.
- Run real tmux QA for enriched socket dashboard rendering.
- Review, QA, document, commit, and push the loop branch.

## Result

Completed.

- `Roster_index.entry` now carries optional description plus domain and tag
  lists.
- `Dashboard_model.agent` now has optional roster metadata.
- `Dashboard_model.enrich_with_roster` applies metadata after model creation,
  so socket snapshots stay actor-scoped and independent of local roster files.
- Agent rows now show compact roster hints such as `QA/testing`.
- The selected preview shows full metadata, including source and tags.
- `tactl dashboard render --roster-index ...` and
  `tactl dashboard render-socket --roster-index ...` both enrich dashboard
  output.
- Tests cover index parsing, model enrichment, state dashboard rendering, and
  socket dashboard rendering with roster metadata.
- QA launched a disposable tmux workspace, rendered through the socket with a
  roster index, and verified metadata plus live QA preview.

## Next Loop Candidate

Load agent markdown frontmatter for richer metadata than `index.json` currently
provides, then use it for dashboard detail panels and future pipeline views.
