# Iteration 2026-05-22-24: Dashboard Frontmatter Metadata

## Brief

Loop 23 made the dashboard roster-native from generated `index.json`. Loop 24
loads local agent markdown frontmatter as a second metadata source so the
dashboard can show richer agent profile detail while keeping the socket
protocol and actor redaction boundary unchanged.

## Acceptance Criteria

- Add a small OCaml frontmatter reader that mirrors the TypeScript indexer:
  leading `---` only, flat keys only, inline arrays, CRLF support, and nested
  YAML ignored.
- Extend `Roster_index.entry` with profile fields: model, complexity,
  compatible runtimes, version, author, and isolation.
- Enrich only local roster entries with safe relative paths and matching
  frontmatter `name` values.
- Keep dashboard enrichment client-side after local model creation or socket
  snapshot decode.
- Render richer metadata in the selected preview without widening compact agent
  rows.
- Cover parser behavior, enrichment safety, state CLI rendering, and socket CLI
  rendering with tests.
- Run deterministic tests, live tmux QA, independent review, and independent
  QA before commit.

## Result

Completed.

- Added `Roster_frontmatter`, a bounded shallow markdown frontmatter reader.
- `Roster_index.enrich_from_frontmatter` now overlays flat local markdown
  fields only for `source = "local"` entries whose path is safe and whose
  frontmatter `name` matches the index entry.
- `tactl dashboard render --roster-index` and
  `tactl dashboard render-socket --roster-index` load markdown metadata from
  paths rooted at the index file directory.
- Dashboard previews now include profile, compatibility, version, author,
  isolation, and role-description lines when present.
- Socket snapshots remain unchanged; enrichment still happens after the
  actor-scoped state/runtime payload is decoded by the CLI.
- Fixtures now include local agent markdown files for state and socket dashboard
  tests.
