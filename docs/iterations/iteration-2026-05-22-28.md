# Iteration 2026-05-22-28: Focusable Pipeline Topology

## Brief

Loop 27 added the pure dashboard runner boundary for a future MIAOU page. Loop
28 adds a pure pipeline topology layer and makes the pipeline section focusable
through the existing dashboard interaction model, while still avoiding a hard
MIAOU dependency.

## Acceptance Criteria

- Add a pure topology module with typed node ids, node movement, contract state,
  and declared ACL edges.
- Keep `pipeline_role` frontmatter as node contract metadata only; do not infer
  workflow edges from natural-language role text.
- Filter declared ACL edges to actor-visible dashboard nodes so redacted agents
  do not leak through topology rendering.
- Add `Pipeline` focus to dashboard interaction.
- Make `Tab` cycle `Agents -> Pipeline -> Workspaces -> Agents`.
- Make `p` focus the pipeline and `Up`/`Down` traverse visible topology nodes
  across workspaces while keeping the selected preview in sync.
- Preserve existing dashboard output where possible, keeping the
  `Pipeline overview` section and ACL disclaimer.
- Cover topology extraction, movement, focused rendering, interaction focus,
  CLI key replay, width limits, and socket render regressions.
- Run review, QA, live tmux smoke, commit, and push the loop branch.

## Result

Completed.

- Added `Dashboard_topology` with private `node_id`, explicit contract state,
  declared ACL edge rows, and wraparound node movement.
- Added `Dashboard_model.topology` to derive topology from the actor-visible
  dashboard model.
- Added `Dashboard_model.focus` to render a focused pipeline marker without
  changing socket payloads.
- Extended `Dashboard_interaction` with `Pipeline` focus, `p/P`, and a
  three-way `Tab` cycle.
- Pipeline `Up`/`Down` now moves the selected workspace/agent globally across
  visible topology nodes, so preview selection follows the topology focus.
- Rendered topology now labels edge categories as `declared-acl` and
  `structured-workflow`, with structured workflow explicitly empty until a
  future structured workflow schema exists.
- Added deterministic unit, interaction, model, and CLI coverage.
