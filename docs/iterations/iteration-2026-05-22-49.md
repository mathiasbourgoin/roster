# Iteration 2026-05-22-49: Visible Launch Profile

## Brief

Loop 49 moves the Enter-first start flow closer to the Herdr-simple target by
making the selected detached agent show the launch profile and command before
the operator presses Enter.

## Product Constraint

Starting an agent should not feel like trusting hidden JSON. TA should keep the
flow at pick agent, press Enter, while showing enough launch intent to make the
action clear.

## Acceptance Criteria

- Configured agent commands are preserved in durable TA state snapshots.
- Legacy snapshots without a command remain readable and show an unknown
  launch command instead of failing.
- Dashboard models expose the command to static and MIAOU renderers.
- Detached MIAOU action labels show profile and compact shell-safe command
  text.
- Static and MIAOU detail views show the launch command.
- Attached-agent refresh labels remain short and unchanged.
- Start preflight rejects stale state/config launch drift, including command,
  configured cwd, env, startup prompt, workspace root, and tmux session.

## Result

Completed.

- Added `Launch_profile`, a shared core helper for launch profile and
  shell-safe command labels.
- Threaded configured command intent from workspace config into state,
  snapshots, dashboard models, static layout, and MIAOU widgets.
- Updated MIAOU detached actions from `Enter Start lead` to labels such as
  `Enter Start lead | Codex | 'codex'`.
- Added coverage for command load, snapshot roundtrip, legacy snapshot default,
  shell-safe profile labels, static dashboard launch text, and MIAOU action
  labels.
- Added preflight coverage for stale launch drift and redaction coverage for
  write-only action targets.
- Resolved workspace roots consistently during config-backed state creation, so
  state preflight can reject workspace root/session drift without breaking the
  normal config path.
- Canonicalized config paths lexically before state bootstrap and launch
  planning, and added a regression for the default `.harness/ta.json`
  Enter-start path.
- Redacted drift error payloads so stale-start failures do not echo launch
  commands, env, startup prompts, workspace roots, or tmux sessions.
