# Iteration 2026-05-23-57: Import-Safe Index Tests

## Brief

Loop 57 removes a recurring review and commit hazard: root `npm test` was
rewriting `index.json` because `build-index` executed at module import time.

## Product Constraint

TA development loops should not carry unrelated generated registry diffs. A
test command must be safe to run during review and QA without changing project
state unless it is explicitly a generation command.

## Acceptance Criteria

- Importing `build-index` for tests does not build or write `index.json`.
- `npm test` passes without changing the existing `index.json` hash.
- `npm run build:index` still writes an index when invoked explicitly.
- Existing helper exports remain importable by tests.
- TA TUI tmux smoke still passes for loop policy.

## Result

Completed.

- Guarded the `build-index` CLI entrypoint with `require.main === module`.
- Added a regression test that imports the compiled module in a temporary
  workspace and verifies no `index.json` is written.
- Verified explicit `build:index` still produces an output file when requested.
