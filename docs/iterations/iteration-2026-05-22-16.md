# Iteration 2026-05-22-16: Runtime Snapshot Cache

## Brief

Add a typed runtime snapshot layer that observes attached tmux panes from a TA
state snapshot and emits cacheable JSON for the future MIAOU/TUI dashboard.

## Acceptance Criteria

- Keep persisted state and live runtime observations separate.
- Capture live, missing, and unattached pane states for every agent in state.
- Verify attached pane ids against persisted tmux pane identity before marking
  them live.
- Capture bounded pane preview lines without making the UI poll tmux directly.
- Add a `tactl runtime snapshot` command that prints JSON or writes a cache
  file with owner-only permissions.
- Cover runtime snapshot collection and CLI behavior with tests.
- Run real tmux QA against a disposable launched workspace.
- Review, QA, document, commit, and push the loop branch.

## Result

Completed.

- Added `Runtime_snapshot`, a typed observation model with workspace/agent
  grouping, configured agent status, optional pane id, pane liveness, and
  preview lines.
- Changed `Tmux.Capture_pane` to target the general tmux target type so native
  `%pane_id` values from launch state can be captured directly.
- State snapshots now persist the expected workspace tmux session and each
  launch-discovered pane's native tmux identity (`session_id` and `window_id`).
  Runtime snapshots verify that identity before capturing preview text.
- Added `tactl runtime snapshot STATE`, with `--lines` and optional
  `--output`, producing JSON suitable for a dashboard cache. Output files are
  written atomically with mode `0600`.
- Added unit coverage for live/missing/unattached pane observations, session
  mismatch rejection, preview blank-row trimming, and exact JSON shape.
- Added CLI coverage for stdout JSON, output-file JSON, owner-only output mode,
  and invalid preview line counts above and below the accepted range.
- QA launched a disposable two-agent tmux workspace, collected a runtime
  snapshot from persisted pane ids, and verified two live panes plus captured
  `lead-ready` and `qa-ready` preview text under pane-identity validation.
- Independent reviewer and QA rechecks passed after pane-identity remediation.

## Next Loop Candidate

Expose runtime snapshots over the Unix socket API. This keeps the dashboard on a
single local control plane instead of shelling out to `tactl runtime snapshot`.
