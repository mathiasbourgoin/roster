# Iteration 2026-05-23-58: Compact Launch Panel

## Brief

Loop 58 keeps TA aligned with the goal of being as simple as Herdr while
surfacing TA's stronger roster/workspace features.

## Product Constraint

The detached-agent first screen should read like a launcher, not a diagnostics
dump. Runtime state, pane ids, and raw capabilities remain useful after an
agent is attached, but before launch the operator needs the target, profile,
authority, and access model.

## Acceptance Criteria

- Detached selected agents render a compact `Launch` panel.
- The launch panel shows workspace, agent, roster identity, profile, authority,
  and read/write access.
- Raw `Capabilities` and diagnostics rows do not appear in the detached launch
  panel.
- Attached agents still keep their `Agent detail` diagnostics.
- The footer remains the primary `Enter Start` or `Enter Refresh` action.

## Result

Completed.

- Replaced detached launch diagnostics rows with compact `Roster` and `Access`
  rows.
- Kept authority visible as a product-level create/connect signal.
- Preserved attached-agent diagnostics and the existing pinned launcher footer.
