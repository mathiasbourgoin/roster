# Iteration 2026-05-22-9: Supervised Launch Plan

## Brief

Prepare supervised tmux launching without spawning long-running agents yet. This
loop defines deterministic launch metadata from workspace config so future
runtime code can create sessions, attach panes, and audit the process against a
stable plan.

## Acceptance Criteria

- Add a pure launch-plan module that validates config before planning.
- Assign deterministic planned pane IDs and tmux targets for each workspace
  agent.
- Resolve effective working directories without touching the filesystem.
- Add `tactl launch plan CONFIG`.
- Cover core plan metadata and CLI output with tests.
- Run review, QA, and tmux smoke before commit.

## Result

Completed.

- Added `Launch_plan`, a pure supervised tmux launch metadata module.
- Added deterministic planned pane IDs using a non-colliding `%` separator.
- Added per-workspace-session tmux targets as `session:0.N`.
- Added config-directory-aware root and cwd resolution for CLI-loaded configs.
- Added `tactl launch plan CONFIG` with optional roster-index validation.
- Added tests for metadata, deterministic rendering, multi-workspace plans,
  dot-containing IDs, env, startup prompts, cwd resolution, and roster failure.
- Review initially found config-directory, tmux target, and pane-id collision
  blockers. All were fixed and re-reviewed with no code blockers.
- QA passed OCaml build/tests/format/lint, plain and roster-aware launch plan,
  roster-aware validation, tmux smoke, and root roster tests.
