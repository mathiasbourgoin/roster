# Iteration 2026-05-22-10: Supervised Tmux Launch Start

## Brief

Turn launch plans into a bounded tmux execution path. This loop adds command
generation and a guarded `tactl launch start` command with a dry-run mode so QA
can inspect exact tmux operations before creating sessions.

## Acceptance Criteria

- Add tmux adapter support for split windows, layout selection, and pane-target
  send-keys.
- Add a launch runtime that builds tmux commands from `Launch_plan.t`.
- Add `tactl launch start CONFIG` and `--dry-run`.
- Refuse to launch if a target tmux session already exists.
- Cover command generation and dry-run output with tests.
- Run review, QA, and tmux smoke before commit.

## Result

Completed.

- Added tmux adapter support for target addresses, split windows, layout
  selection, literal prompt sending, and shell-copyable command rendering.
- Added `Launch_runtime` to turn `Launch_plan.t` into tmux command sequences.
- Added `tactl launch start CONFIG` and `--dry-run`.
- Real launch refuses duplicate sessions and existing sessions before creating
  panes.
- Failure cleanup now kills only sessions created by the current launch process.
- Startup prompts use literal `send-keys -l` plus a separate Enter, preventing
  prompt text such as `C-c` or `Enter` from being interpreted as tmux keys.
- QA verified real tmux session creation, two pane captures, prompt literal
  behavior, duplicate-session dry-run validation, no stderr leaks, and cleanup
  ownership.
- Review initially found cleanup ownership, dry-run validation, and prompt
  literal-mode issues. All were fixed and re-reviewed with no code blockers.
