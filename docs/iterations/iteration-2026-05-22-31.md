# Iteration 2026-05-22-31: Target-Specific Edge Commands

## Brief

Loop 30 introduced focused-edge affordances. Loop 31 makes those affordances
target-specific so the future MIAOU page can cycle among visible targets on a
declared ACL edge, keep preview selection aligned, and expose explicit source
and target pane-focus intents without adding socket mutations.

## Acceptance Criteria

- Track the selected target for a focused pipeline edge.
- Add keyboard replay for target cycling inside the selected edge.
- Preserve selected edge target across refresh when it remains visible.
- Render a selected-target marker in focused-edge preview metadata.
- Add typed `Focus_pane` intents for source and readable selected targets.
- Keep target read/focus intents gated by read permission.
- Keep write intents actor-bound to the edge source.
- Cover pure affordance rendering, interaction target cycling, refresh
  preservation, write-only target suppression, CLI key replay, and live
  tmux/socket behavior.
- Run review, QA, live tmux smoke, commit, and push the loop branch.

## Result

Completed.

- Added `Focus_pane` as a typed read intent in `Dashboard_edge_affordance`.
- `Dashboard_edge_affordance.render_preview` can mark a selected target.
- `Dashboard_model.edge_affordance` accepts a selected target and emits actions
  for that target instead of every target on the edge.
- `Dashboard_interaction` now tracks `selected_edge_target`.
- `]` and `[` cycle visible targets within the currently selected edge while
  keeping the preview panel on the selected target.
- Refresh preserves the selected edge target when it is still present.
- Write-only targets keep actor-bound write intents but suppress read-class
  preview and pane-focus intents.
- Added deterministic tests for target cycling, refresh preservation,
  write-only target permission gating, selected target rendering, and CLI
  `--key p --key Right --key ]`.
- Verified multi-target behavior through a live tmux/socket smoke.

## Next Loop Candidate

Add a non-mutating dashboard action inspector/exporter so future TUI bindings,
MCP tools, and tests can request the focused edge action list as structured
data instead of parsing rendered preview text.
