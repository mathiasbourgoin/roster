# Iteration 2026-05-22-50: Two-Axis Launcher Footer

## Brief

Loop 50 makes the normal MIAOU dashboard behave more like a launcher:
Left/Right selects a workspace, Up/Down selects an agent, and Enter starts or
refreshes the selected agent.

## Product Constraint

TA should be as simple as Herdr at the first screen. Workspace and roster power
should be visible without making the operator manage focus modes before the
first start.

## Acceptance Criteria

- The default non-pipeline TUI path uses Left/Right for workspace selection.
- The default non-pipeline TUI path uses Up/Down for agent selection.
- Pipeline edge navigation still uses Right/Left as before.
- The selected workspace, agent, launch profile, and Enter action are pinned in
  a full-width MIAOU footer.
- Short and narrow terminal layouts keep the launcher action visible.
- Tiny terminal layouts keep the launcher footer pinned even when body content
  is clipped away.
- Enter-start behavior remains unchanged after switching workspaces.

## Result

Completed.

- Added focus-independent launcher movement for Workspaces and Agents modes.
- Added a pinned MIAOU launch footer with selected target, profile, action, and
  compact command.
- Removed the duplicated top action from the main panel so short terminals keep
  preview and detail content visible while the footer owns the action.
- Clipped body content before appending the footer so tiny terminals cannot
  push the launcher action out of the frame.
- Updated key hints to explain the workspace and agent axes.
- Added pure interaction and MIAOU headless coverage for workspace switching
  and the launch footer.
