# Iteration 2026-05-23-54: Navigation-Only Launcher Sidebar

## Brief

Loop 54 removes duplicate action/source text from the MIAOU sidebar so the
first screen reads more like a launcher and less like a diagnostics report.

## Product Constraint

The sidebar should answer only two questions in the normal path: which
workspace and which agent. The launch panel explains the selected launch target,
and the pinned footer owns the primary Enter action.

## Acceptance Criteria

- The sidebar renders workspace and agent navigation only.
- Detached launch views no longer duplicate `Enter Start <agent>` in the
  sidebar.
- The launch panel still shows workspace/source context.
- The pinned footer still shows the selected target and complete `Enter Start`.
- Attached agents still render preview plus `Agent detail` and complete
  `Enter Refresh`.
- Runtime, socket, state, and config behavior remain unchanged.

## Result

Completed.

- Simplified `sidebar_text` to render only workspace and agent navigation.
- Removed the now-unused selected-action sidebar helper.
- Updated harness tests to assert footer-owned start action and no duplicate
  sidebar start text.
- Preserved attached-agent diagnostics through existing preview/detail paths.
