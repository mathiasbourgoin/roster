# Iteration 2026-05-22-51: Launcher Capability Powers

## Brief

Loop 51 makes roster-native authority visible in the same pinned launcher
footer introduced in loop 50.

## Product Constraint

TA should not hide important roster powers behind config files. Privileged
agents such as `tech-lead` and `recruiter` should visibly differ from ordinary
agents without making the first screen more complex.

## Acceptance Criteria

- Selected agents with `create-agent` and/or `connect-agents` capabilities show
  a compact capability segment in the MIAOU launch footer.
- Ordinary agents do not show capability powers in the footer.
- The capability segment appears before the command so it is visible at normal
  terminal widths.
- Existing Enter start/refresh behavior is unchanged.

## Result

Completed.

- Added a compact `Authority create+connect` footer segment for agents with
  both TA product capabilities.
- Added single-capability variants for future partial-capability agents.
- Added MIAOU headless coverage for privileged harness agents and ordinary QA
  agents.
