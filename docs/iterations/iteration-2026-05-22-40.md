# Iteration 2026-05-22-40: Simple Local TUI Start

## Brief

Loop 40 reduces the normal human start path toward the Herdr bar: open `ta`,
select an agent, press `s`. The local TUI can now start the selected agent
without a manually started socket server when it has a trusted config path.

## Acceptance Criteria

- Preserve the loop 39 socket safety contract: `start-agent` socket clients may
  not send config paths or actor identities.
- Let the local `ta` process call the same trusted start implementation used by
  the socket server when no `--socket` is supplied.
- Auto-create `.ta-state.json` from a discovered config when state is missing,
  without overwriting an existing state snapshot.
- After a successful selected-agent start, attach the captured tmux pane and
  mark the agent `running`.
- Cover the flow with MIAOU headless and isolated tmux execution.

## Result

Completed.

- Exposed `Socket_api.start_agent` as the shared trusted start path.
- `ta` now starts selected agents directly when a state dashboard has a config
  path and no socket override.
- `ta` now bootstraps `.ta-state.json` from a discovered config before rendering
  the dashboard, so a config-only workspace becomes state-backed immediately.
- Successful selected-agent starts now persist the pane and transition status to
  `running`.
- The `ta --help` man text now documents pressing `s` in the TUI.

## Validation

- `opam exec -- dune build @all @runtest`
- MIAOU headless direct-start test with a temp config and no pre-existing state:
  pressing `s` bootstraps `.ta-state.json`, starts the pane, and renders
  `Runtime LIVE`, `Status running`, and `attached | r refresh`.

## Review Notes

The local direct path deliberately uses the selected agent as the actor. This is
only for the in-process human TUI path. The socket protocol still rejects
client-supplied config, roster index, and actor for `start-agent`; server-side
socket starts still require `tactl socket serve --config ... --actor ...`.

Generated `index.json` remains unstaged.

## Next Loop Candidate

Turn startup into a true one-command app experience: when `ta` finds a config
but no state, make the TUI the first screen by default, add an obvious start
footer/control affordance, and remove remaining help text that implies users
must manually run `tactl state save` before using the TUI.
