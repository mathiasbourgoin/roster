# Iteration 2026-05-22-45: Enter Primary Action

## Brief

Loop 45 makes the normal MIAOU TUI workflow closer to Herdr: select a workspace,
select an agent, press `Enter`. Detached agents start; attached agents refresh.
The richer roster, harness, privilege, and capability details remain visible
without making startup feel like configuration work.

## Acceptance Criteria

- Detached selected agents show `Enter Start <agent>`.
- Pressing `Enter` on a detached selected agent starts it through the existing
  direct or socket-backed start path.
- Attached selected agents show `Enter Refresh | attached <pane>`.
- Pressing `Enter` on an attached selected agent refreshes the dashboard and
  does not attempt another start.
- `s` remains a direct start shortcut for detached agents.
- Pressing `s` on an attached selected agent is guarded with a clear stale
  message instead of creating another pane.
- Help, quickstart, root man text, and MIAOU key hints lead with `Enter`.
- No create/connect mutation flows, command-profile selectors, or config forms
  are added in this loop.

## Result

Completed.

- Added contextual primary action handling in `ta_tui`: `Enter`, `Return`, and
  `C-m` all start detached agents or refresh attached agents.
- Guarded the legacy `s` start shortcut so attached agents cannot be started a
  second time from the selected-agent action.
- Updated MIAOU action labels from `s Start` / `r Refresh` to
  `Enter Start` / `Enter Refresh`.
- Updated startup guide, `ta` help, and `tactl` root help to document the
  Enter-first flow.
- Added headless MIAOU tests for Enter start, Enter refresh, and guarded
  attached-agent `s`.

## Review Notes

This loop intentionally avoids new configuration screens. It moves the primary
action to the obvious key and keeps advanced actions as shortcuts. Restart,
resume, create-agent, and connect-agent flows should be explicit future actions
instead of overloading attached-agent Enter behavior.

Generated `index.json` remains unstaged.
