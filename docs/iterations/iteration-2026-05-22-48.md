# Iteration 2026-05-22-48: Anchored Preview Focus

## Brief

Loop 47 added `v` as a live-preview focus toggle. Loop 48 makes that focus
predictable: it is anchored to the selected live workspace, agent, and dashboard
view that requested it.

## Product Constraint

TA should stay as simple as Herdr while using roster-aware features. Preview
focus is a direct inspection mode, not a hidden global mode that follows the
operator around the dashboard.

## Acceptance Criteria

- Pressing `v` on a live selected agent focuses that agent preview.
- Pressing `v` on a detached or missing selected agent does not arm hidden
  focus state.
- Manual refresh preserves preview focus only while the same live agent and
  view remain selected.
- Moving selection or switching dashboard views clears preview focus.
- Returning to a previously focused live agent after moving away does not
  resurrect stale focus.
- Enter start/refresh behavior remains unchanged.

## Result

Completed.

- Replaced the MIAOU preview-focus boolean with an anchored page state carrying
  workspace, agent, and dashboard focus.
- Reconciled the anchor after refresh/start/key updates so focus only survives
  when the same live selection remains active.
- Added headless MIAOU coverage for refresh preservation, selection clearing,
  stale-focus prevention after moving away and back, and pipeline-view clearing.
