# Iteration 2026-05-22-20: Socket Dashboard Refresh Source

## Brief

Move the dashboard refresh path from direct state-file reads toward the local
control plane. This loop adds an actor-scoped dashboard snapshot endpoint and a
deterministic `render-socket` command, so the future MIAOU runner can refresh
state and live previews through the same socket boundary used by managed agent
sessions.

## Acceptance Criteria

- Add a typed dashboard snapshot that combines saved state with authorized
  runtime previews.
- Scope runtime preview collection to self-read and workspace `read` links for
  the requesting actor.
- Reject unknown actors and invalid preview line bounds.
- Add a `dashboard-snapshot` socket command.
- Add `tactl dashboard render-socket` with actor, selection, key replay, width,
  and preview-line options.
- Cover snapshot parsing, socket command behavior, render-socket output, and
  authorization with tests.
- Run real tmux QA through the socket-backed dashboard path.
- Review, QA, document, commit, and push the loop branch.

## Result

Completed.

- Added `Dashboard_snapshot`, a typed boundary carrying saved `State_store.t`
  plus actor-authorized `Runtime_snapshot.t` data.
- Added `dashboard-snapshot` to the Unix socket protocol and API.
- Added `tactl dashboard render-socket`, which requests the snapshot over a
  socket and renders it through `Dashboard_interaction`.
- Runtime previews remain actor-scoped: actors see their own panes and panes
  reachable through workspace `read` links.
- Saved dashboard state is also actor-redacted: actors see only workspaces where
  they exist, visible agents, visible links, and no audit history.
- Socket dashboard responses are aggregate-bounded against the socket line cap,
  trimming preview payloads before returning an oversized response.
- Added unit tests and socket CLI tests for actor scoping, JSON round-tripping,
  bounded JSON encoding, required actor validation, unknown actor errors, and
  render-socket selection.
- QA launched a disposable two-agent tmux workspace, rendered the dashboard over
  a socket, selected the QA pane with `Down`, and verified the live QA preview
  plus QA actor state/runtime redaction.

## Next Loop Candidate

Add socket-backed refresh ticks to the dashboard interaction loop, including
refresh error state, stale snapshot rendering, and deterministic CLI key replay
