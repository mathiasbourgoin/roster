# Iteration 2026-05-22-11: Persist Launch Pane Metadata

## Brief

Close the runtime loop between supervised tmux launch and TA state snapshots by
recording the native tmux pane ids created for each launched roster agent.

## Acceptance Criteria

- Query real tmux pane ids after a successful launch.
- Return typed agent-to-pane attachments from the launch runtime.
- Add `tactl launch start --state STATE CONFIG` to write those attachments into
  the state snapshot with audit events.
- Preflight state compatibility before launching when `--state` is used.
- Clean up launch-created sessions if pane querying or state update fails.
- Add deterministic unit coverage and real tmux QA evidence.

## Result

Completed.

- Added `Tmux.Display_pane_id`, which runs `display-message -p -t TARGET
  "#{pane_id}"`.
- Changed `Launch_runtime.run` to return typed attachments containing
  workspace id, agent id, planned pane id, native pane id, and tmux target.
- Added `Launch_state` for pure preflight and attachment application.
- Added `tactl launch start --state STATE CONFIG`; successful launches now
  update state snapshots with `Pane_attached` audit events.
- Added cleanup for failures after launch, including state write failures after
  the sessions have already been created.
- Added deterministic runtime tests through an injectable tmux runner, plus
  pure state transition tests and CLI option coverage.
- Reviewer found state-update cleanup and query-path test gaps; both were
  remediated before commit.
- QA passed unit tests, root checks, live tmux pane persistence, and state
  update failure cleanup.

## Next Loop Candidate

Make launch target discovery robust for non-default tmux `base-index` and
`pane-base-index` settings before relying on positional targets in larger
workspace launches.
