# Iteration 2026-05-22-8: State Detail View

## Brief

Add a detailed state inspection command before starting daemon or TUI work. The
command should expose enough information for scripts, QA, and future TUI
screens to inspect agent status, tmux panes, ACL links, and recent audit events
without reading snapshot JSON by hand.

## Acceptance Criteria

- Add a state detail renderer that includes summary, workspaces, agents, panes,
  ACL links, and recent audit events.
- Add `tactl state show STATE` with an `--audit-limit` option.
- Keep the renderer deterministic and covered by command-level tests.
- Run review, QA, and tmux smoke before commit.

## Result

Completed.

- Added `State_store.describe` for deterministic detailed rendering of state
  snapshots.
- Added `tactl state show STATE` with `--audit-limit`.
- The detail view includes summary, workspace metadata, agents, status, panes,
  ACL links, and recent audit events.
- Negative audit limits are rejected at the CLI boundary.
- Command tests now assert exact output for the default audit limit, limit `2`,
  limit `0`, and negative-limit rejection.
- Review and QA passed after tightening the audit-limit validation and exact
  output coverage.
