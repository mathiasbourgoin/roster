# Iteration 2026-05-21-2: Roster-Aware Validation

## Brief

TA configs should not only validate internal workspace shape. They should also
be able to check that every `roster_agent` points at an actual agent in
agent-roster's `index.json`.

## Acceptance Criteria

- Add a minimal `index.json` reader that filters `component_type = "agent"`.
- Add validation that rejects unknown `roster_agent` values when a roster index
  is supplied.
- Add `tactl validate --roster-index index.json`.
- Add tests for roster index parsing and unknown roster agents.
- Run review and QA before commit.

## Result

Completed.

- Added `Roster_index`, a minimal reader for generated `index.json` that keeps
  only `component_type = "agent"` entries.
- Added `Workspace_config.validate_with_roster`.
- Added `tactl validate --roster-index INDEX CONFIG`.
- Added tests for roster index parsing and unknown `roster_agent` values.
- Review found no blocking correctness issues.
- QA passed positive and negative roster-index CLI checks plus tmux smoke.

## Follow-Up

Add an automated CLI-level test harness for `tactl validate --roster-index`;
this iteration verified the CLI manually and through QA.
