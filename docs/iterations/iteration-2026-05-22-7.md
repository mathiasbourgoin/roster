# Iteration 2026-05-22-7: State Mutation Commands

## Brief

Make persisted state snapshots operational by adding small mutation commands for
the runtime fields that already exist in the pure state store. This loop still
avoids a daemon; it gives scripts, QA, and the future socket layer a stable
command contract.

## Acceptance Criteria

- Add `tactl state set-status STATE --workspace W --agent A --status S` with an
  optional audit actor and required reasons for blocked/failed status.
- Add `tactl state attach-pane STATE --workspace W --agent A --pane PANE` with
  an optional audit actor.
- Persist mutations atomically through `State_file.save`.
- Validate IDs and status values before mutating snapshots.
- Cover successful mutations and invalid status handling with command tests.
- Run review, QA, and tmux smoke before commit.

## Result

Completed.

- Added `tactl state set-status` for persisted agent status mutation with typed
  workspace/agent/actor validation.
- Added `tactl state attach-pane` for persisted pane attachment, including
  native tmux pane ids such as `%77`.
- Added `State_store.status_of_string` with mandatory reasons for blocked and
  failed statuses.
- Added `State_file.update`, which locks the state path around load-mutate-save
  so concurrent CLI mutations do not silently drop audit events.
- Expanded command tests to assert persisted status, pane, and audit kind; added
  a semantic failure test proving unknown actors leave snapshots unchanged.
- Added a forked regression test for concurrent `State_file.update` calls.
- Review and QA initially found the `%77` pane-id gap and an unlocked
  read-modify-write risk; both were fixed and re-verified.
- QA passed OCaml build/tests/format/lint, manual `%77` mutation commands,
  tmux smoke execution, and root roster tests.
