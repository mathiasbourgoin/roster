# Iteration 2026-05-22-36: Replace Notty With Miaou

## Brief

Loop 35 introduced a first full-screen dashboard loop, but it used Notty as a
bootstrap. Mathias explicitly rejected that direction and required TA to use
MIAOU. Loop 36 removes the Notty runner, makes `miaou-tui` the terminal UI
dependency, and keeps dashboard content in a pure layout boundary that can grow
into richer MIAOU widgets.

## Acceptance Criteria

- Remove the Notty dependency from TA.
- Add the `miaou-tui` opam package and required MIAOU runner libraries.
- Implement `ta` full-screen mode as a MIAOU page.
- Preserve `--tui=auto|always|never` behavior and static non-TTY output.
- Support `MIAOU_DRIVER=headless` for JSON-driven automation.
- Add layout tests for selected-agent, pipeline-edge, clipping, narrow width,
  and driver-owned footer behavior.
- Run review, automated tests, tmux QA, commit, push, and open the next loop
  branch.

## Result

Completed.

- Replaced `ta_tui.ml` with a `Miaou_core.Direct_page` backed by
  `Dashboard_runner`.
- `ta` now launches through `Miaou_runner_tui.Runner_tui`.
- Added `Dashboard_tui_layout`, a bounded layout model with header, sidebar,
  main region, optional footer, and responsive width calculation.
- Updated `dune-project` and opam metadata from `notty` to `miaou-tui`.
- Added MIAOU headless CLI coverage and pure layout tests.
- Updated startup/help copy to say the TUI uses `miaou-tui`.
- Verified real tmux startup/navigation/quit with MIAOU term and matrix
  backends.

## Environment Note

The `octez-setup` switch cannot currently solve `miaou-tui` because it pins
Octez's `cohttp.5.3.1~octez`, while Miaou's opam metadata requires the newer
`cohttp-eio >= 6.2.1` stack. Verification used the `/home/mathias/dev/miaou`
switch, where `miaou-tui` installs cleanly. The next infrastructure loop should
give `ocaml/agent-manager` its own local opam switch or CI setup so `dune exec
ta` works without relying on the broader Octez switch.
