# Iteration 2026-05-22-14: ACL-Checked Socket Mutations

## Brief

Extend the Unix socket control plane with audited state mutations while keeping
the trust boundary explicit and enforcing workspace write permissions.

## Acceptance Criteria

- Add socket requests for `set-status` and `attach-pane`.
- Reuse existing `State_store` transitions and `State_file.update`.
- Require an actor for socket mutations.
- Allow self-writes and configured `write` edges only.
- Keep socket paths private and owner-only.
- Review, QA, document, commit, and push the loop branch.

## Result

Completed.

- Extended `Socket_protocol.request` with typed `Set_status` and `Attach_pane`
  variants.
- Added `tactl socket request ... set-status` and `attach-pane` CLI support.
- Socket mutations now require an actor. The actor may update itself, or update
  another agent only when the workspace graph grants `Permission.Write`.
- The socket server rejects group/other-accessible socket directories and
  chmods the socket path to `0600` after bind.
- Added tests for mutation success, unauthorized actor rejection, missing actor
  rejection, regular-file socket path refusal, shared-directory refusal, and
  socket file mode.
- Reviewer found the missing socket trust boundary and ACL checks; both were
  remediated and reviewer recheck passed.
- QA verified self-mutation success, unauthorized cross-agent rejection, state
  hash stability after denial, persisted audit events, and cleanup.

## Next Loop Candidate

Add socket supervised launch requests using the existing launch runtime and
state update flow, with explicit cleanup and review coverage for partial launch
failure cases.
