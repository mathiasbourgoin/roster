# Iteration 2026-05-22-34: Default Startup And Help

## Brief

Loop 33 added terminal viewport bounds, but the user-facing entrypoint still
failed the first-run experience: `dune exec ta` printed a bootstrap placeholder
and required the user to already know the state/config flags. Loop 34 makes TA
start with workspace defaults and turns help/quickstart output into an actual
startup guide.

## Acceptance Criteria

- Make bare `ta` resolve default workspace state/config paths before falling
  back to instructions.
- Prefer saved state over config so live dashboards win when both exist.
- Render a config-backed dashboard when only a config is available, instead of
  showing a placeholder.
- Keep the default path workspace-enabled while supporting this source tree's
  bundled example config.
- Add `tactl quickstart` with source-tree and installed command flows.
- Expand `ta --help` and `tactl --help` so the start path is visible without
  external docs.
- Cover empty workspace, default config, resolver, quickstart, and help output
  with tests.
- Run review, QA, tmux smoke or dry-run evidence, commit, and push the loop
  branch.

## Result

Completed.

- Added typed startup path candidates for state and config defaults.
- `ta` now uses the first existing default state/config when flags are omitted.
- If only a config exists, `ta` builds an in-memory state and renders the
  dashboard immediately, with detached panes.
- If no defaults exist, `ta` prints a concrete quickstart instead of the old
  "No state provided" placeholder.
- Added shared quickstart text and the `tactl quickstart` command.
- Added `ta` and `tactl` man/help sections that describe default startup,
  source-tree commands, and the current MIAOU adapter status.
- Split startup guidance into real workspace setup through `.harness/ta.json`
  and bundled example setup through `ta.json`, so the example's `root "."`
  remains correct when copied unchanged.
- Verified `dune exec ta` from `ocaml/agent-manager` opens the dashboard from
  `examples/ta.example.json`.
