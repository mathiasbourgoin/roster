# Iteration 2026-05-22-27: Dashboard Runner Boundary

## Brief

Loop 26 added the static pipeline overview. Loop 27 establishes the pure
dashboard event runner that a future MIAOU page can call. Local research found
MIAOU checkouts and the expected `PAGE_SIG`/runner surface, but `miaou-tui` is
not installed in the current switch and Cabal is an execution backend rather
than the TUI layer, so this loop keeps TA dependency-free.

## Acceptance Criteria

- Add a typed dashboard runner boundary for key and tick events.
- Reuse `Dashboard_interaction`, `Dashboard_refresh_cadence`, and injected
  dashboard refresh sources.
- Preserve selected-agent state across successful refreshes.
- Record refresh successes and failures in the cadence state.
- Stop automatic refresh work after quit.
- Route existing CLI dashboard key replay through the runner where a refresh
  source exists, without changing rendered dashboard output.
- Cover tick, manual refresh, failure, quit, cadence rendering, and opaque key
  behavior with deterministic tests.
- Run review, QA, tmux smoke, commit, and push the loop branch.

## Result

Completed.

- Added `Dashboard_runner` with private key values, typed key/tick events, a
  GADT event boundary, typed refresh outcomes, and a pure step/run API.
- Added `of_interaction` so existing selected dashboard state can enter the
  runner without being reset.
- Rewired `ta` and `tactl` dashboard key replay through the runner when refresh
  callbacks are available.
- Added deterministic runner tests for first tick refresh, interval wait,
  manual refresh preserving selection, refresh failure, quit suppression,
  cadence render lines, key roundtrips, and unknown key no-op behavior.
- Kept MIAOU as a planned integration point instead of adding a hard dependency
  before the current opam switch can build it directly.

## Next Loop Candidate

Use the runner boundary to add a concrete MIAOU page adapter or, if dependency
installation remains blocked, add a richer focusable pipeline topology model
that the page can render once the adapter lands.
