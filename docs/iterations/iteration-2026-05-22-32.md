# Iteration 2026-05-22-32: Dashboard Action Export

## Brief

Loop 31 made focused pipeline edge commands target-specific. Loop 32 turns the
same typed affordances into structured JSON so future MIAOU bindings, MCP tools,
and tests can inspect available read/write intents without scraping rendered
dashboard text or mutating tmux/socket state.

## Acceptance Criteria

- Add a pure, documented action export boundary.
- Serialize focused-edge endpoints, targets, permissions, selection, and GADT
  socket intents as stable JSON.
- Expose state-file and socket-backed CLI inspectors that reuse dashboard key
  replay and refresh semantics.
- Preserve actor-scoped redaction and actor-bound write intent gating.
- Keep the exporter non-mutating: no tmux focus changes, pane writes, socket
  mutations, or state snapshot writes.
- Cover no-focus, focused-edge, selected target, lines propagation, actor write
  suppression, write-only target, state CLI, and socket CLI behavior.
- Run review, QA, live tmux smoke, commit, and push the loop branch.

## Result

Completed.

- Added `Dashboard_action_export`, a pure schema-versioned export module for
  dashboard interaction state.
- Added JSON helpers to `Dashboard_edge_affordance` for endpoints, targets,
  typed intents, and actions.
- Added `tactl dashboard actions STATE` and `tactl dashboard actions-socket`
  for read-only structured inspection after optional key replay.
- `dashboard actions --actor` can expose actor-bound write intents from state
  snapshots.
- Reused existing dashboard refresh sources, so `--key r` refreshes before
  exporting without adding socket protocol commands, and export JSON includes
  refresh status so stale snapshots are explicit.
- Socket action export keeps write-only targets as pane-less placeholders,
  allowing write intents without exposing terminal previews.
- Added unit and CLI tests for JSON shape, selected target cycling,
  `--lines` propagation, socket actor requirement, write-only target export,
  stale refresh status, and permission gating.
- Verified the socket exporter against a disposable live tmux workspace and
  confirmed the state snapshot hash did not change.
