# Iteration 2026-05-22-25: Dashboard Pipeline Role Detail

## Brief

Loop 24 loaded flat markdown frontmatter into roster-backed dashboard detail.
Loop 25 adds typed support for nested `pipeline_role` blocks so each selected
agent can show its orchestration contract without turning natural-language
metadata into a premature graph model.

## Acceptance Criteria

- Add a typed OCaml `pipeline_role` reader with strong wrappers for
  `triggered_by`, `receives`, `produces`, and `human_gate`.
- Parse only the leading markdown frontmatter block, find a top-level
  `pipeline_role:`, read indented scalar subfields, stop at the next top-level
  key, ignore unknown fields, and require all four fields.
- Keep duplicate subfields last-write-wins and reject empty required fields.
- Thread `pipeline_role` through local safe frontmatter enrichment only when
  the markdown `name` matches the index entry.
- Render pipeline detail only in the selected-agent preview panel.
- Keep compact rows, socket payloads, and socket redaction unchanged.
- Add deterministic tests for parser behavior, unsafe enrichment skips, local
  dashboard rendering, socket dashboard rendering, and raw socket snapshot
  metadata absence.
- Run review, QA, live tmux smoke, commit, and push the loop branch.

## Result

Completed.

- Added `Roster_pipeline_role` with typed private wrappers.
- `Roster_index.entry` now carries optional typed pipeline role metadata.
- Local frontmatter enrichment reads the markdown file once and overlays both
  flat metadata and nested pipeline role data after safe path/name checks.
- Dashboard preview detail now shows `Pipeline`, `Receives`, `Produces`, and
  `Human gate` lines for enriched agents.
- Raw `dashboard-snapshot` responses remain roster-metadata-free and
  actor-scoped.
- Review and QA passed after a parser fix for duplicate empty subfields and
  stronger scoped-render assertions.
