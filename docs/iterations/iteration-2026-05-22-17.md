# Iteration 2026-05-22-17: Socket Runtime Snapshots

## Brief

Expose loop 16's cacheable runtime snapshot model through the local Unix socket
API so future MIAOU/TUI clients can use one control plane instead of shelling
out to `tactl runtime snapshot`.

## Acceptance Criteria

- Add a typed actor-scoped `runtime-snapshot` socket request.
- Reuse `Runtime_snapshot.collect` and its pane-identity verification.
- Enforce the same preview line bounds as the CLI.
- Bound preview bytes so an authorized socket response cannot overflow the
  one-line socket transport.
- Add `tactl socket request --workspace W --agent A --actor X --lines N
  runtime-snapshot`.
- Enforce self-read or workspace `read` links before returning pane previews.
- Cover socket protocol/CLI behavior with tests.
- Run real tmux QA through socket request.
- Review, QA, document, commit, and push the loop branch.

## Result

Completed.

- Added `Socket_protocol.Runtime_snapshot { workspace; agent; actor; lines }`.
- `Socket_api.execute` loads the served state snapshot, collects the runtime
  snapshot for the requested agent, and returns compact JSON in the socket
  success payload.
- Runtime snapshot requests now require an actor and authorize either self-read
  or a workspace `Permission.Read` edge before exposing terminal previews.
- The socket path reuses `Runtime_snapshot.max_preview_lines`, rejecting line
  counts below 1 and above 200.
- Runtime snapshots cap individual preview line bytes and total preview bytes
  before JSON serialization.
- Added `tactl socket request --workspace W --agent A --actor X --lines N
  runtime-snapshot`.
- Added socket CLI tests for runtime snapshot JSON, missing actor, unauthorized
  actor, bad `--lines`, and over-cap `--lines`.
- Added runtime snapshot tests for long-line truncation and total preview byte
  caps after review identified the socket transport risk.
- Reviewer and QA blockers around unscoped preview exposure were remediated by
  requiring workspace/agent/actor scope and read authorization.
- QA launched a disposable two-agent tmux workspace, requested scoped runtime
  snapshots through `socket serve --once`, and verified self-read, read-edge
  access, live pane previews, persisted pane identities, and denial for an actor
  without read permission.

## Next Loop Candidate

Start the MIAOU/TUI dashboard foundation using the socket runtime snapshot model
as its read path. Keep it Herdr-quality from the first screen: dense workspace
navigation, fast keyboard movement, readable live agent state, and pane preview
confidence.
